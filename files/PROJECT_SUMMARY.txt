- Load your inspection and thermal reports
- Extract structured observations
- Merge data and detect conflicts
- Analyze root cause and severity
- Generate a professional DDR report

================================================================================
THE 4-STAGE ARCHITECTURE
================================================================================

STAGE 1: Structured Data Extraction
   ‚Üì Extract observations from each document separately
   ‚Üì JSON schema enforcement
   ‚Üì No summarization, only explicit facts

STAGE 2: Data Cleaning & Logical Merging
   ‚Üì Group observations by area
   ‚Üì Detect conflicts automatically
   ‚Üì Mark missing data
   ‚Üì Remove duplicates

STAGE 3: Root Cause & Severity Reasoning
   ‚Üì Analyze merged observations only
   ‚Üì Determine probable root cause
   ‚Üì Assign severity with reasoning
   ‚Üì No invention of facts

STAGE 4: Final DDR Generation
   ‚Üì Generate client-friendly report
   ‚Üì Simple language, no jargon
   ‚Üì All required sections
   ‚Üì Explicit missing information list

================================================================================
WHY THIS APPROACH?
================================================================================

TRADITIONAL APPROACH (‚ùå):
- Single LLM call to summarize both documents
- High hallucination risk
- Hard to validate
- Mixes extraction with reasoning

THIS APPROACH (‚úÖ):
- Separates extraction from reasoning
- Each stage validates previous output
- Conflicts detected automatically
- Missing data explicitly handled
- Easy to debug and improve

================================================================================
RELIABILITY FEATURES
================================================================================

‚úì Hallucination Prevention
  - "Do NOT invent facts" in every prompt
  - JSON schema enforcement
  - Multi-stage validation

‚úì Missing Data Protocol
  - Always write "Not Available" instead of guessing
  - Explicitly list all missing information
  - No assumptions made

‚úì Conflict Detection
  - Automatic detection of contradictions
  - Temperature vs. description consistency checks
  - Conflicts mentioned in report

‚úì Data Quality Checks
  - Source attribution for all observations
  - Completeness assessment per area
  - Duplicate removal
  - Confidence scoring

================================================================================
EXAMPLE OUTPUT
================================================================================

The system generates two files:

1. ddr_report.json - Structured JSON with all data
2. ddr_report_formatted.txt - Human-readable report

Report includes:
- Property Issue Summary
- Area-wise Observations (with temps)
- Probable Root Cause
- Severity Assessment (Low/Medium/High + reasoning)
- Recommended Actions
- Additional Notes (conflicts, etc.)
- Missing Information (explicitly listed)
